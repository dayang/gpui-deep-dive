# 前言

## 这本书写给谁

你至少用 GPUI 写过几个界面，能用 `div()`、`Label`、`cx.spawn()` 完成基本工作。但你心中有一些挥之不去的疑问：

- `Context` 为什么要写成 `cx: &mut Context<Self>`？那个 `Self` 参数除了让你感觉在写 Java 泛型之外还有什么意义？
- 为什么我的元素状态偶尔会丢失？滚动位置重置、展开状态折叠——无声无息，没有任何 panic。
- `Entity` 和 `View` 到底是什么关系？为什么有时候说"实体"，有时候说"视图"？
- `cx.notify()` 调了，为什么界面没变？或者说，它变了，但是在下一个 `if` 之后才变的？
- 异步操作为什么必须用 `cx.update()` 而不能直接 `await` 后接着写？

这本书的目标就是回答这些问题。我们不会教你如何写一个 hello world 或计数器应用——那是 README 和博客文章的工作。本书假设你已经会"用" GPUI，现在需要"理解" GPUI。

## 这本书不做什么

- **不是 API 文档**。本书不会罗列每个函数签名。当你需要查某个方法的参数类型时，去读 `docs.rs/gpui`。
- **不是迁移指南**。本书基于 GPUI 0.2.2 源码分析，但不会追踪每个版本的 breaking change。
- **不是设计文档**。我们会讨论"为什么这么设计"，但最终权威永远在源码。如果书中推断与源码表现不一致，以源码为准。

## 阅读建议

**每章都配有源文件引用**，格式为 `（src/app.rs:123-456）`，方便你跳转到源码对照阅读。建议你在 IDE 中打开 gpui 的源码。

全书七部分，从心智模型到具体机制，再到案例分析，由浅入深。第一、二部分是基础——理解了帧循环、所有权模型和实体系统，后面的机制分析才能对上号。

某些章末尾会有"如果你只记住一件事"——那一句话就是该章的核心结论。时间紧的话，至少读这个。
