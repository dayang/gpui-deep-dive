# 第 15 章：键盘动作系统

键盘是 GPUI 设计哲学中最核心的输入方式。GPUI 是"键盘优先"的框架——所有 UI 操作都通过 Action 系统可绑定。这与许多 GUI 框架"鼠标优先、键盘是补充"的思路相反。

> 源文件参考：`src/action.rs`、`src/keymap.rs`、`src/key_dispatch.rs`、`docs/key_dispatch.md`

## Action 的定义

GPUI 提供了两种定义 Action 的方式：

### 方式一：`actions!` 宏（单元结构体）

```rust
actions!(
    editor,
    [
        MoveUp,
        MoveDown,
        Newline,
        Backspace,
        Delete,
        SelectAll,
    ]
);
```

展开后生成：

```rust
#[derive(Clone, PartialEq, Action)]
pub struct MoveUp;
#[derive(Clone, PartialEq, Action)]
pub struct MoveDown;
// ...
```

### 方式二：`#[derive(Action)]`（带数据的 Action）

```rust
#[derive(Action, Clone, PartialEq)]
#[action(namespace = "editor")]
pub struct Move {
    pub direction: Direction,
    #[action(default)]
    pub select: bool,
}
```

这种形式支持：
- **JSON 序列化**：keymap 文件中可以这样绑定：`"cmd-k": ["editor::Move", { "direction": "Up", "select": true }]`
- **默认值**：`#[action(default)]` 字段在 JSON 中可省略
- **弃用**：`#[action(deprecated = "Use Tab instead")]` 标记旧 Action

## Action trait 内部

```rust
pub trait Action: 'static + Send + Clone + PartialEq {
    fn boxed_clone(&self) -> Box<dyn Action>;
    fn partial_eq(&self, other: &dyn Action) -> bool;
    fn name(&self) -> &'static str;       // "editor::Move"
    fn name_for_type() -> &'static str;    // 类型级别的名字
    fn build(value: Option<serde_json::Value>) -> Result<Box<dyn Action>, ActionBuildError>;
    fn action_json_schema() -> Option<serde_json::Value>;
    fn deprecated_aliases() -> &'static [&'static str];
}
```

全部通过 derive 宏自动生成，你不需要手动实现。

## Action 注册（inventory）

GPUI 使用 `inventory` crate 实现**编译期注册**：

```rust
// derive 宏展开（简化）
#[derive(Action)]
pub struct Move { ... }

// 宏生成：
inventory::submit! {
    ActionRegistryEntry {
        name: "editor::Move",
        builder: |json| {
            let action: Move = serde_json::from_value(json)?;
            Ok(Box::new(action) as Box<dyn Action>)
        },
    }
}
```

`inventory::collect!` 在程序启动时收集所有注册的 Action。这意味着：
- 你不需要手动调用 `register_action!`
- 所有 Action 在编译时就已注册
- 可以在 keymap JSON 中按名称引用任何 Action

## Keymap：绑定按键到 Action

Keymap 使用 JSON 格式：

```json
[
  {
    "context": "Editor",
    "bindings": {
      "up": "editor::MoveUp",
      "down": "editor::MoveDown",
      "cmd-shift-up": ["editor::Move", { "direction": "Up", "select": true }],
      "enter": "editor::Newline",
      "backspace": "editor::Backspace"
    }
  },
  {
    "context": "Editor && mode == vim",
    "bindings": {
      "h": "editor::MoveLeft",
      "j": "editor::MoveDown",
      "k": "editor::MoveUp",
      "l": "editor::MoveRight"
    }
  }
]
```

### KeyContext 匹配

在元素上设置 key context：

```rust
div()
    .key_context("Editor")
    .on_action(cx.listener(|this, _: &MoveUp, cx| {
        // ctrl-p 或 up 都会触发这里
    }))
```

KeyContext 支持布尔表达式：`"Editor && mode == vim"`。当前元素的 context 栈决定哪些绑定生效。

### 匹配优先级

当一个按键被按下时：

1. 收集当前元素栈的所有 key context
2. 从 keymap 文件中按顺序匹配
3. 第一个匹配的 context → binding 生效
4. 如果没有匹配，keystroke 被丢弃或传递给全局处理器

## 完整的分发流程

```
按键事件（来自平台）
  │
  ├─ 1. 全局 keystroke 拦截器（on_keystroke）
  │     如果拦截器调用了 cx.stop_propagation() → 结束
  │
  ├─ 2. Keymap 解析
  │     keystroke → Keymap::binding_for_keystroke(contexts)
  │     → Option<Box<dyn Action>>
  │
  ├─ 3. DispatchTree 上的 dispatch_action
  │     Capture 阶段 → Bubble 阶段
  │     每个节点的 on_action 回调被调用
  │     cx.stop_propagation() 可中止
  │
  └─ 4. 全局 keystroke 观察器（keystroke observers）
        只读——不能停止传播，只用于记录/监控
```

## on_action 的注册

```rust
impl Render for MyView {
    fn render(&mut self, _window: &mut Window, cx: &mut Context<Self>) -> impl IntoElement {
        div()
            .on_action(cx.listener(|this, action: &MoveUp, cx| {
                // 处理 MoveUp
            }))
            .on_action(cx.listener(|this, action: &MoveDown, cx| {
                // 处理 MoveDown
            }))
    }
}
```

`cx.listener()` 生成了一个闭包，它持有 `WeakEntity<Self>`，在 dispatching 时执行。

## NoAction：取消默认绑定

```rust
// 在 keymap 中
{
    "context": "Editor",
    "bindings": {
        "ctrl-w": null  // null = NoAction = 取消默认关闭窗口的行为
    }
}
```

`NoAction` 是一个特殊的哨兵 Action——当 keymap 返回 `NoAction` 时，dispatch 被跳过，但事件被消费（不会触发全局处理器）。

## Keymap 的加载

```rust
// App 创建时
let keymap = Keymap::new(
    include_str!("../assets/keymaps/default.json"),  // 内置默认
    vec![custom_keymap],  // 用户自定义
);
```

多个 keymap 文件被合并——后加载的覆盖先加载的同名绑定。这让用户可以在不修改框架源码的情况下自定义快捷键。

---

**如果你只记住一件事：** GPUI 用 `actions!` 或 `#[derive(Action)]` 定义 Action，用 JSON keymap 文件绑定按键，通过 KeyContext 表达式实现上下文敏感的按键映射。
