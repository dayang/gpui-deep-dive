# 第 15 章：键盘动作系统

GPUI 是"键盘优先"的框架——所有 UI 操作都通过 Action 系统可绑定。本章基于源码精确定义 Action trait、注册机制和 Keymap。

> 源文件参考：`src/action.rs`、`src/keymap.rs`、`src/key_dispatch.rs`

## Action 的定义

GPUI 提供了两种定义 Action 的方式：

### 方式一：`actions!` 宏（单元结构体）

```rust
actions!(
    editor,
    [
        MoveUp,
        MoveDown,
        Newline,
    ]
);
```

展开后生成（`src/action.rs:26-28`）：

```rust
#[derive(Clone, PartialEq, Default, Debug, Action)]
pub struct MoveUp;
#[derive(Clone, PartialEq, Default, Debug, Action)]
pub struct MoveDown;
// ...
```

### 方式二：`#[derive(Action)]`（带数据的 Action）

```rust
#[derive(Action, Clone, PartialEq)]
#[action(namespace = "editor")]
pub struct Move {
    pub direction: Direction,
    #[action(default)]
    pub select: bool,
}
```

支持 JSON 序列化用于 keymap 绑定。

## Action trait 完整定义

```rust
// src/action.rs:117-173
pub trait Action: Any + Send {
    fn boxed_clone(&self) -> Box<dyn Action>;
    fn partial_eq(&self, action: &dyn Action) -> bool;
    fn name(&self) -> &'static str;
    fn name_for_type() -> &'static str where Self: Sized;
    fn build(value: serde_json::Value) -> Result<Box<dyn Action>> where Self: Sized;
    fn action_json_schema(_: &mut schemars::SchemaGenerator) -> Option<schemars::Schema>
        where Self: Sized { None }
    fn deprecated_aliases() -> &'static [&'static str] where Self: Sized { &[] }
    fn deprecation_message() -> Option<&'static str> where Self: Sized { None }
    fn documentation() -> Option<&'static str> where Self: Sized { None }
}
```

关键点：
- 超 trait 是 `Any + Send`（不是 `Clone + PartialEq`）。
- `build()` 参数不是 `Option<Value>`，是 `serde_json::Value`（必须传入至少 `{}`）。
- 8 个方法，其中 4 个有默认实现。
- `dyn Action` 通过 `as_any()` 方法（`src/action.rs:185`）向下转型为具体类型。

## Action 注册（inventory）

GPUI 使用 `inventory` crate 实现**编译期注册**。derive 宏生成类似这样的代码：

```rust
// `#[derive(Action)]` 展开效果（简化）
inventory::submit! {
    MacroActionBuilder(MacroActionData {
        name: "editor::Move",
        type_id: TypeId::of::<Move>(),
        build: |json| Ok(Box::new(serde_json::from_value::<Move>(json)?)),
        json_schema: None,
        deprecated_aliases: &[],
        deprecation_message: None,
        documentation: None,
    })
}
```

`inventory::collect!` 在程序启动时收集所有注册的 Action。实际注册器类型是 `MacroActionBuilder(MacroActionData)`。

## Keymap 和 KeyBinding

```rust

pub struct Keymap {
    bindings: Vec<KeyBinding>,

impl Keymap {

```

`Keymap::new()` 接受预解析的 `Vec<KeyBinding>`（而非原始 JSON 字符串）。在应用中通过 `cx.bind_keys()` 绑定（`src/app.rs:1696` — `App::bind_keys`）。

## 完整的分发流程

```
按键事件（来自平台）
  │
  ├─ 1. 全局 keystroke 拦截器（cx.intercept_keystrokes）
  │     如果调用了 cx.stop_propagation() → 结束
  │
  ├─ 2. 确定 dispatch_path
  │     focused → focus_path 对应的 dispatch_path
  │     无焦点 → 使用根节点
  │
  ├─ 3. dispatch_tree.dispatch_key(pending, keystroke, &dispatch_path)
  │     ├─ 沿 dispatch_path 收集 context_stack
  │     ├─ keymap.bindings_for_input(&input, &context_stack)
  │     └─ 返回 DispatchResult { bindings, pending, to_replay, context_stack }
  │
  ├─ 4. 对每个 binding，沿 dispatch_path 分发：
  │     Capture 阶段 → Bubble 阶段
  │     每个节点的 action_listeners 被调用
  │     callback: Fn(&dyn Any, DispatchPhase, &mut Window, &mut App)
  │
  └─ 5. 全局 keystroke 观察器（cx.observe_keystrokes）
        只读——不能停止传播
```

## 在元素上注册 action listener

```rust
impl Render for MyView {
    fn render(&mut self, window: &mut Window, cx: &mut Context<Self>) -> impl IntoElement {
        div()
            .key_context("Editor")
            .on_action(cx.listener(MyView::handle_move_up))
    }
}

impl MyView {
    fn handle_move_up(&mut self, _: &MoveUp, _window: &mut Window, _cx: &mut Context<Self>) {
        // 处理 MoveUp
    }
}
```

`cx.listener()` 生成一个闭包，内部持有 `WeakEntity<Self>`。Action listener 的回调签名为 `Fn(&E, &mut Window, &mut App)`（不是 `Fn(&mut Self, &E, &mut Context<Self>)` 的三参数形式）。

## 全局 action 处理

```rust
// App 级别注册
cx.on_action(|app: &Quit, cx: &mut App| {
    cx.quit();
});

// 全局 keystroke 拦截
cx.intercept_keystrokes(|keystroke, cx| {
    // 在 dispatch 之前触发，可 cx.stop_propagation()
});

// 全局 keystroke 观察
cx.observe_keystrokes(|keystroke, cx| {
    // dispatch 之后触发，只读
});
```

## KeyContext 匹配

```rust
div()
    .key_context("Editor")
    .on_action(cx.listener(Editor::undo))
```

在 `paint` 阶段，`.key_context("Editor")` 调用 `window.set_key_context(context)`（`src/window.rs:3333`），在当前 DispatchNode 上设置 key context。

KeyContext 支持布尔表达式：`"Editor && mode == vim"`（`src/keymap/context.rs`）。当前元素栈的所有 key context 决定了哪些 key binding 生效。

---

**如果你只记住一件事：** `Action` 的超 trait 是 `Any + Send`（共 8 个方法）。通过 `inventory` 编译期注册。key binding 通过 `cx.bind_keys()` 注册，dispatch 通过 `DispatchTree::dispatch_key()` 沿 dispatch_path 匹配并分发。
