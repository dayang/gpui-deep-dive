# 第 13 章：跨平台渲染器对比

GPUI 支持三个 GPU 渲染后端：macOS 上的 Metal、Windows 上的 DirectX 11、以及作为跨平台备选的 Blade。本章对比它们的实现差异和 GPUI 如何抽象。

> 源文件参考：
> - `src/platform/mac/metal_renderer.rs` —— macOS Metal 渲染器
> - `src/platform/windows/directx_renderer.rs` —— Windows DirectX 11 渲染器
> - `src/platform/blade/blade_renderer.rs` —— 跨平台 Blade 渲染器
> - `build.rs` —— shader 编译配置

## 渲染器在架构中的位置

```
Window::draw()
  └── Scene::finish() → BatchIterator
        │
        └── PlatformRenderer::render(batches)
              │
              ├── MetalRenderer (macOS, 默认)
              ├── BladeRenderer (macOS 可选 / Linux 默认)
              └── DirectXRenderer (Windows)

统一输入：Scene::batches()
统一输出：framebuffer（在平台窗口中呈现）
```

三个渲染器都消费相同的数据结构（`PrimitiveBatch`），产生相同的结果（屏幕像素）。它们只是"如何提交给 GPU"的方式不同。

## Shader 对比

| | Metal (macOS) | DirectX 11 (Windows) | Blade (跨平台) |
|---|---|---|---|
| **Shader 语言** | MSL (Metal Shading Language) | HLSL (High-Level Shading Language) | WGSL (WebGPU Shading Language) |
| **源文件** | `src/platform/mac/shaders.metal` | 内嵌在 `directx_renderer.rs` 中 | `src/platform/blade/shaders.wgsl` |
| **编译时机** | 构建时 (xcrun) | 构建时 (fxc.exe) | 构建时验证 (naga)；运行时编译 |
| **输出** | `shaders.metallib` | 内嵌为字节数组 | 运行时由 blade 编译 |

### Metal Shader 编译流程（build.rs）

```bash
# 1. cbindgen 生成 C header（scene.h）—— 导出 Rust 结构体到 C 可见
cbindgen → scene.h

# 2. shaders.metal 中 #include "scene.h" —— MSL shader 知道 Rust 结构体的内存布局
# 3. xcrun metal 编译为 .air (Apple Intermediate Representation)
xcrun -sdk macosx metal shaders.metal -o shaders.air

# 4. xcrun metallib 打包为 .metallib (Metal Library)
xcrun -sdk macosx metallib shaders.air -o shaders.metallib
```

关键点：**cbindgen 确保 Rust 的 `Quad`/`Shadow` 等结构体和 Metal shader 中的结构体内存布局一致**。这让 CPU 侧和 GPU 侧能共享数据结构而无需手动维护对齐。

### DirectX Shader 编译流程

```bash
# 对每种 shader 类型分别编译
fxc.exe /T vs_5_0 /E vs_main quad.hlsl /Fo quad_vs.cso
fxc.exe /T ps_5_0 /E ps_main quad.hlsl /Fo quad_ps.cso
# 编译结果嵌入为 Rust 字节数组
```

DX shader 有多个变体：`quad`, `shadow`, `path_rasterization`, `path_sprite`, `underline`, `monochrome_sprite`, `polychrome_sprite`, `emoji_rasterization`。

### Blade Shader 编译

```
shaders.wgsl → build.rs 用 naga 解析验证 → 运行时由 blade-graphics 编译为对应平台的原生 shader
```

WGSL 是 WebGPU 标准的一部分。Blade 将其编译为目标平台的原生 shader 语言（macOS 上编译为 MSL，Windows 上编译为 HLSL，Linux 上通过 SPIR-V 到 GLSL/Vulkan）。

## 主要 Shader 类型

不管哪个平台，这些 shader 做同样的事：

| Shader | 顶点输入 | 片段处理 |
|--------|---------|---------|
| **Quad** | 矩形顶点 + 颜色/圆角/border | 背景填充、边框、圆角裁剪 |
| **Shadow** | 矩形顶点 + 模糊半径 + 颜色 | 高斯模糊（采样周围像素） |
| **MonochromeSprite** | 矩形 + UV 坐标 + 颜色 | 采样 atlas 纹理 × 颜色 |
| **PolychromeSprite** | 矩形 + UV 坐标 | 采样 atlas 纹理（保留原色） |
| **Underline** | 线段顶点 + 颜色 + 波浪参数 | 直线或波浪线填充 |
| **Path** | 三角剖分后的顶点 | 使用 lyon 的 tessellation 结果填充 |

## Metal 渲染器细节

```rust
// src/platform/mac/metal_renderer.rs
pub struct MetalRenderer {
    device: metal::Device,
    command_queue: metal::CommandQueue,
    pipeline_states: HashMap<PipelineKey, metal::RenderPipelineState>,
    atlas_textures: Vec<metal::Texture>,
    // ...
}
```

Metal 使用 `CAMetalLayer` 作为渲染目标。每一帧：

1. 从 `CAMetalLayer` 获取 `nextDrawable`
2. 创建 `MTLCommandBuffer`
3. 对每个 batch 创建 `MTLRenderCommandEncoder`
4. 上传顶点/统一变量数据
5. `drawPrimitives`（三角形）
6. `commitCommandBuffer` → `presentDrawable`

## DirectX 渲染器细节

```rust
// src/platform/windows/directx_renderer.rs
pub struct DirectXRenderer {
    device: ID3D11Device,
    context: ID3D11DeviceContext,
    swap_chain: IDXGISwapChain,
    shaders: ShaderLibrary,
    // ...
}
```

DX11 使用传统的 swap chain + 立即模式渲染：

1. 清除 render target view
2. 映射顶点缓冲区（`Map` / `Unmap`）
3. 设置 shader、常量缓冲区、纹理
4. `DrawIndexed`

与 Metal 的重要区别：DX11 的 `Map/Unmap` 需要在每次绘制前更新缓冲区（因为没有 Metal 的 `setVertexBytes` 等价方法）。

## Blade 渲染器

Blade 是 gpui 自己的跨平台 GPU 抽象。它的设计目标不是成为一个通用 GPU 库，而是精确覆盖 GPUI 需要的功能。

```rust
// src/platform/blade/blade_renderer.rs
pub struct BladeRenderer {
    context: blade::Context,
    // ...
}
```

Blade 在 macOS 上封装 Metal，在 Windows 上封装 Vulkan/DirectX，在 Linux 上封装 Vulkan。它的 shader 使用 WGSL，在构建时由 naga 验证并翻译为各平台原生格式。

Blade 主要用于：
- **Linux（Wayland + X11）**：因为 Linux 没有单一"标准"GPU API。
- **macOS 可选（`macos-blade` feature）**：替代 Metal shader。

## 各平台特性对比

| 特性 | Metal | DirectX 11 | Blade |
|------|-------|------------|-------|
| **平台** | macOS only | Windows only | 跨平台 |
| **渲染 API 版本** | Metal 3 | DX11 (FL 11.0) | 可变 |
| **纹理格式** | MTLPixelFormatBGRA8Unorm | DXGI_FORMAT_B8G8R8A8_UNORM | BGRA8 |
| **Vsync** | CADisplayLink | DWM (composition) | 平台相关 |
| **HiDPI** | contentsScale | DPI awareness | 平台相关 |
| **透明窗口** | ✅ (vibrant/blurred) | ⚠️ 部分支持 | ❌ |
| **Shader 热重载** | ❌ (预编译) | ❌ (预编译) | ✅ (runtime_shaders) |

## build.rs 中的配置（开发者视角）

```toml
# Cargo.toml
[features]
macos-blade = []      # macOS 上用 Blade 代替 Metal
runtime_shaders = []  # Metal shader 运行时编译（开发时有用，避免每次改 shader 重编译）
```

- **默认**：macOS=Metal，Windows=DX11，Linux=Blade
- **`macos-blade`**：macOS 上也用 Blade（放弃 Metal 的某些优化，换取统一代码路径）
- **`runtime_shaders`**：开发时开启，允许修改 shader 后无需重新编译整个项目

---

**如果你只记住一件事：** 三个渲染器都消费同一个 Scene::batches() 输出，区别在于如何提交给 GPU。shader 用各自平台的语言实现相同逻辑。cbindgen/naga 确保 CPU 和 GPU 侧的数据结构对齐。
