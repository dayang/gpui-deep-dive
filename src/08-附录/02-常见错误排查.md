# 附录 B：常见错误排查

## 编译错误

### `cx` does not live long enough

**症状**：
```
error[E0597]: `cx` does not live long enough
  --> src/my_view.rs:42:18
```

**原因**：你试图把 `Context<'a, T>` 传给需要 `'static` 的闭包。

**解决**：

```rust
// 错误
cx.spawn(|mut cx| async move {
    cx.update(|cx| { ... })  // ❌ `cx` 是 Context，不是 'static
});

// 正确——GPUI 的 cx.spawn 自动转为 AsyncApp
cx.spawn(|mut cx: AsyncApp| async move {
    cx.update(|cx| { ... })  // ✅ cx 现在是 AsyncApp
}).detach();
```

### `cx` is not `Send`

**症状**：
```
error: future cannot be sent between threads safely
```

**原因**：你在 `async move {}` 中持有 `Context<'a, T>`（不是 Send）。

**解决**：用 `cx.spawn()` 而非手写 async block。

```rust
// 错误
let future = async move {
    cx.notify();  // ❌ cx 不是 Send
};

// 正确
cx.spawn(|mut cx| async move {
    // AsyncApp 是 Send
    cx.update(|cx| cx.notify()).ok();
}).detach();
```

### `type annotations needed for `cx.update(...)``

**症状**：编译器无法推断 `update` 闭包的类型。

**解决**：显式标注 `Context<T>` 的类型。

```rust
cx.update(|cx: &mut Context<MyView>| {
    // 现在编译器知道 cx 的类型了
    cx.notify();
}).ok();
```

## 运行时错误

### `already borrowed: BorrowMutError` (panic)

**症状**：
```
panicked at 'already borrowed: BorrowMutError'
```

**原因**：在 observer/subscriber 回调中尝试借入 App 时发生循环借用。

**常见触发场景**：

```rust
// Bug：observer 中 notify 同一实体
cx.observe(&entity, |this: &mut Self, _entity, cx| {
    cx.notify();  // ❌ 此时 entity 已被借出（观察它的变化）
});
```

**解决**：使用 `cx.defer()` 将 notify 推迟到当前效果循环之后。

```rust
cx.observe(&entity, |this: &mut Self, _entity, cx| {
    cx.defer(|cx| {
        cx.notify();  // ✅ 在下一轮 flush 中执行
    });
});
```

### 元素状态丢失

**症状**：滚动位置、展开状态或输入内容在某个交互后重置。

**原因**：`GlobalElementId` 路径变化导致 element state 失配。详见第 10 章。

**快速检查清单**：
- [ ] 是否在条件渲染中包裹了需要保持状态的元素？
- [ ] 父元素的 ID 是否在帧间改变了？
- [ ] 列表项的 key 是否稳定（不在重排时变化）？
- [ ] 是否在渲染函数的顶层就决定了所有 ID？

**常见修复**：

```rust
// 修复前（状态丢失）
if self.show_panel {
    div().id("panel").child(ScrollArea { ... })  // ❌
}

// 修复后（状态保持）
div()
    .id("panel")
    .visible(self.show_panel)
    .child(ScrollArea { ... })  // ✅
```

### Entity 过早释放

**症状**：`WeakEntity::upgrade()` 返回 `None` 或操作 entity 时 panic "entity not found"。

**原因**：所有 `Entity<T>` 强引用已释放。

**调试**：

```rust
// 打印引用计数
let counts = entity.ref_counts();
eprintln!("entity: {}, weak: {}", counts.entity_count(), counts.weak_count());
```

**解决**：在需要 entity 保持的地方持有 `Entity<T>`（不是 `WeakEntity<T>`）。

## 逻辑错误

### notify() 后界面不更新

**症状**：调用了 `cx.notify()` 但界面没有变化。

**可能原因**：

1. **`notify()` 在错误的实体上调用**。

```rust
// Bug：notify 了错误的对象
this.other.notify(cx);  // ❌ other 和当前视图无关
// 正确：应该 notify 当前 Context 所在的实体
cx.notify();  // ✅
```

2. **Window 没有被标记为待刷新**。

检查 `flush_effects` 链是否正常。在某些情况下（如在 `AsyncApp::update` 中操作已被关闭的窗口），通知可能被丢弃。

3. **cached_style 阻止了 render**。

```rust
// 如果使用了 .cached()，确认 cached_style 的依赖字段确实变化了
let view = cx.new(|_| MyView { ... }).cached(style_refinement);
```

### subscribe 不生效

**症状**：`cx.emit()` 被调用了但 subscriber 没有执行。

**原因**：

1. **Subscription 被立即 drop**。

```rust
// Bug：Subscription 没有被持有
impl MyView {
    fn init(&mut self, cx: &mut Context<Self>) {
        cx.subscribe(&self.other, |this, event, cx| {
            // ...
        });  // ❌ 返回值被丢弃 → 订阅立即取消
    }
}

// 正确：持有 Subscription
struct MyView {
    _subscription: Subscription,  // ← 必须保持存活
}
```

2. **事件类型不匹配**。

```rust
// subscribe 的事件类型必须与 emit 的类型完全匹配
cx.emit(MyEvent::A);  // emit MyEvent
// subscribe 必须也接收 MyEvent（不是其他类型）
cx.subscribe(&entity, |this, event: &MyEvent, cx| { ... });
```

## 性能问题

### render() 调用过于频繁

**症状**：帧率低，CPU profile 显示大量时间在 `render()` 中。

**可能原因**：

1. **notify 循环**：observer 中 notify → 又触发 render。

**解决**：合并多次通知为一次。在数据变更后批量 notify，而非每次小改动都 notify。

```rust
// 不好
for item in &new_items {
    self.items.push(item.clone());
    cx.notify();  // 每次都触发一次完整的 render 周期
}

// 好
self.items.extend(new_items);
cx.notify();  // 一次通知
```

2. **大数据集不设 key**：参见第 10 章。

### 布局缓慢

**症状**：`request_layout` 阶段耗时过长。

**原因**：Taffy 求解大量节点的布局（特别是深层嵌套 + flexbox 的情况）。

**解决**：
- 减少不必要的嵌套
- 用 `UniformList` 替代 `List`
- 预先计算固定尺寸（用 `request_measured_layout` 替代 `request_layout`）

## 调试技巧

### 打印 EntityId

```rust
eprintln!("EntityId: {:?}", entity.entity_id());
```

### 激活 inspector

```toml
[dependencies]
gpui = { version = "0.2.2", features = ["inspector"] }
```

```rust
// 在应用中
cx.activate_inspector();
```

Inspector 可以让你在运行时浏览元素树、查看布局信息和 entity 状态。

### 启用 leak-detection

```toml
[features]
leak-detection = ["gpui/leak-detection"]
```

```rust
// 程序退出时报告未释放的 Entity
```

### 追踪 flush_effects

在 GPUI 的调试构建中，可以在 `flush_effects()` 开始和结束时加日志，追踪效果队列的长度和类型分布。
