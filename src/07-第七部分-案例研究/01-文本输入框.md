# 第 19 章：手写一个文本输入框

本章基于 `examples/input.rs`（747 行），逐段讲解如何从零实现一个功能完整的文本输入框。它演示了自定义 `Element`、光标/选区管理、IME 支持和剪贴板交互。

> 源文件参考：`examples/input.rs`、`src/input.rs`（`EntityInputHandler` trait）

## 我们要做什么

一个支持以下功能的文本输入框：
- 字符插入和删除
- 光标定位（鼠标点击和键盘方向键）
- 选区（Shift+方向键 扩展选区）
- 剪贴板操作（Ctrl+C / Ctrl+X / Ctrl+V）
- IME 组合输入（中文输入法的 marked text）
- 键盘导航（Home/End, Ctrl+Left/Right 单词级移动）

## 步骤一：理解 examples/input.rs 的整体结构

```
struct InputElement {
    focus_handle: FocusHandle,
    editor: Entity<Editor>,
}

struct Editor {
    text: SharedString,
    selection: Option<Range<usize>>,
    marked_text: Option<(Range<usize>, SharedString)>,
    // ...
}
```

两个实体分工：
- **`Editor`**：数据模型——文本内容、选区、标记文本。
- **`InputElement`**：UI 桥接——持有 `FocusHandle` 和 editor 引用，实现 `Element` trait。

## 步骤二：生命周期管理

输入框需要响应按键。这需要实体间的通信：

```rust
impl InputElement {
    fn new(editor: Entity<Editor>, cx: &mut Context<Self>) -> Self {
        // 订阅 Editor 的变化
        cx.subscribe(&editor, |this: &mut Self, event: &EditorEvent, cx| {
            match event {
                EditorEvent::Changed { ... } => {
                    cx.notify();  // 触发重绘
                }
            }
        });

        Self {
            focus_handle: cx.focus_handle(),
            editor,
        }
    }
}
```

## 步骤三：实现 Element trait

```rust
impl Element for InputElement {
    type RequestLayoutState = TextLayout;  // ← 跨阶段传递文本布局
    type PrepaintState = ();

    fn request_layout(
        &mut self,
        _id: Option<&GlobalElementId>,
        window: &mut Window,
        cx: &mut App,
    ) -> (LayoutId, Self::RequestLayoutState) {
        let editor = self.editor.read(cx);

        // 在 request_layout 阶段准备文本布局
        let text_layout = window.text_system().layout_line(
            &editor.text,
            font_size,
            &[],
        );

        let size = text_layout.size();  // 文本的像素尺寸
        let layout_id = window.request_measured_layout(
            Style::default(),
            size.into(),
        );
        (layout_id, text_layout)
    }

    fn prepaint(
        &mut self,
        _id: Option<&GlobalElementId>,
        bounds: Bounds<Pixels>,
        text_layout: &mut Self::RequestLayoutState,
        window: &mut Window,
        cx: &mut App,
    ) {
        // 注册 hitbox
        window.insert_hitbox(bounds, HitTest::default());
    }

    fn paint(
        &mut self,
        _id: Option<&GlobalElementId>,
        bounds: Bounds<Pixels>,
        text_layout: &mut Self::RequestLayoutState,
        _prepaint: &mut Self::PrepaintState,
        window: &mut Window,
        cx: &mut App,
    ) {
        let editor = self.editor.read(cx);

        // 1. 绘制选区背景（蓝色矩形）
        if let Some(selection) = &editor.selection {
            let selection_bounds = text_layout.bounds_for_range(selection.clone());
            window.paint_quad(PaintQuad {
                bounds: selection_bounds,
                background: Some(highlight_blue),
                ..Default::default()
            });
        }

        // 2. 绘制文本
        window.paint_text(text_layout, bounds.origin, editor.text.clone(), text_color);

        // 3. 绘制光标（闪烁的竖线）
        if self.focus_handle.is_focused(cx) {
            let cursor_pos = text_layout.cursor_position(editor.cursor_offset);
            window.paint_quad(PaintQuad {
                bounds: Bounds::new(
                    bounds.origin + cursor_pos,
                    Size::new(cursor_width, bounds.size.height),
                ),
                background: Some(text_color),
                ..Default::default()
            });
        }
    }
}
```

impl Render for InputElement {
    fn render(&mut self, _window: &mut Window, cx: &mut Context<Self>) -> impl IntoElement {
        div()
            .track_focus(&self.focus_handle)
            .on_action(cx.listener(|this, _: &Backspace, cx| {
                this.editor.update(cx, |editor, cx| {
                    editor.backspace();
                    cx.notify();
                });

## 步骤五：IME 输入处理

IME（Input Method Editor）处理是文本框中较复杂的部分。GPUI 通过 `EntityInputHandler` trait 来管理：

```rust
impl EntityInputHandler for Editor {
    fn text_for_range(
        &mut self,
        range_utf16: Range<usize>,
        cx: &mut Context<Self>,
    ) -> Option<SharedString> {
        // 返回指定 UTF-16 范围对应的文本
        Some(self.text[range_utf16].into())
    }

    fn selected_text_range(
        &mut self,
        cx: &mut Context<Self>,
    ) -> Option<Range<usize>> {
        // 返回当前选区的 UTF-16 范围（px 单位，需要转换为字符偏移）
        self.selection.clone()
    }

    fn marked_text_range(
        &mut self,
        cx: &mut Context<Self>,
    ) -> Option<Range<usize>> {
        // 返回组合输入文本的范围
        self.marked_text.as_ref().map(|(range, _)| range.clone())
    }

    fn replace_text_in_range(
        &mut self,
        range_utf16: Option<Range<usize>>,
        text: SharedString,
        cx: &mut Context<Self>,
    ) {
        // IME 提交文本时的回调
        if let Some(range) = range_utf16 {
            self.text.replace_range(range, &text);
        } else {
            self.text.push_str(&text);
        }
        cx.notify();
    }

    fn replace_and_mark_text_in_range(
        &mut self,
        range_utf16: Option<Range<usize>>,
        new_text: SharedString,
        new_selected_range: Option<Range<usize>>,
        cx: &mut Context<Self>,
    ) {
        // IME 组合输入中（marked text 更新）
        self.marked_text = Some((range_utf16.unwrap_or(0..0), new_text));
        cx.notify();
    }

    fn unmark_text(&mut self, cx: &mut Context<Self>) {
        // 组合输入结束
        self.marked_text = None;
        cx.notify();
    }
}
```

## 步骤六：剪贴板操作

```rust
// 复制
fn copy_selection(&self, cx: &mut Context<Self>) {
    if let Some(selection) = &self.selection {
        let text = self.text[selection.clone()].to_string();
        cx.write_to_clipboard(ClipboardItem::new_string(text));
    }
}

// 粘贴
fn paste(&mut self, cx: &mut Context<Self>) {
    if let Some(item) = cx.read_from_clipboard() {
        if let Some(text) = item.text() {
            self.insert_text(&text, cx);
        }
    }
}
```

## 关键收获

1. **分离数据与 UI**：`Editor` 是 Entity（数据），`InputElement` 是 Element（UI）。通过 `subscribe` 通信。
2. **Element 三阶段的实际应用**：`request_layout` 时创建 TextLayout，`paint` 时用它绘制文本/选区/光标。
3. **IME 需要实现的全部重的方法**：`EntityInputHandler` 的 7 个方法匹配了 macOS NSTextInputClient 的语义。
4. **剪贴板是平台抽象**：通过 `Platform::write_to_clipboard` 和 `read_from_clipboard`。
