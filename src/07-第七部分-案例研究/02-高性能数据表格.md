# 第 20 章：手写一个高性能数据表格

本章基于 `examples/data_table.rs`（474 行），讲解如何构建一个支持 10,000 行数据、自定义排序、滚动条拖拽的数据表格。它展示了 `UniformList`、自定义 `RenderOnce` 组件、和滚动交互的实现。

> 源文件参考：`examples/data_table.rs`、`src/elements/uniform_list.rs`

## UniformList：高性能列表的秘密

普通的 `List` 会为每个可见元素创建完整的 Element，这是 O(n) 的渲染开销。`UniformList` 利用了**所有行高度相同**的约束，做了大幅优化：

```rust
// 内部原理（简化）
struct UniformList {
    item_count: usize,
    item_height: Pixels,
    scroll_offset: Pixels,
    // 只为可见行调用 render——通常是 30-40 行而不是 10,000 行
}

// 可见行范围计算
fn visible_range(&self, viewport_height: Pixels) -> Range<usize> {
    let start = (self.scroll_offset / self.item_height).floor() as usize;
    let visible_count = (viewport_height / self.item_height).ceil() as usize + 1;
    let end = (start + visible_count).min(self.item_count);
    start..end
}
```

从 10,000 个元素减少到 30 次 `render()` 调用——这就是虚拟化列表的威力。

## 步骤一：数据结构

```rust
// 表格模型
struct DataTable {
    rows: Vec<StockRow>,     // 10,000 行数据
    sort_column: SortColumn,
    sort_ascending: bool,
}

struct StockRow {
    symbol: SharedString,
    price: f64,
    change: f64,
    volume: u64,
    market_cap: f64,
}

enum SortColumn {
    Symbol,
    Price,
    Change,
    Volume,
    MarketCap,
}
```

## 步骤二：UniformList 的渲染

```rust
impl Render for DataTable {
    fn render(&mut self, _window: &mut Window, cx: &mut Context<Self>) -> impl IntoElement {
        let sorted_rows = self.sorted_rows();  // 根据当前排序生成有序数据

        div()
            .flex_col()
            .size_full()
            .child(
                // 表头
                div()
                    .flex()
                    .child(self.header_cell("Symbol", SortColumn::Symbol, cx))
                    .child(self.header_cell("Price", SortColumn::Price, cx))
                    .child(self.header_cell("Change", SortColumn::Change, cx))
                    .child(self.header_cell("Volume", SortColumn::Volume, cx))
                    .child(self.header_cell("Market Cap", SortColumn::MarketCap, cx))
            )
            .child(
                // 数据行——使用 uniform_list
                uniform_list(
                    cx.entity().clone(),  // self 的 view handle
                    "data_rows",          // element id（用于 element state）
                    sorted_rows.len(),
                    move |this, range, _window, cx| {
                        // 闭包：为每行生成元素
                        // range = 可见行索引范围
                        let rows = this.read(cx).sorted_rows();
                        range.map(|i| {
                            TableRow::new(rows[i].clone())
                                .into_any_element()
                        }).collect()
                    },
                )
                .flex_grow(1.)
                .track_scroll(cx.entity(), |this, offset, cx| {
                    // 记录滚动偏移
                    // UniformList 自动管理 element state
                })
            )
    }
}
```

## 步骤三：TableRow 作为 RenderOnce 组件

```rust
#[derive(IntoElement)]
struct TableRow {
    data: StockRow,
}

impl RenderOnce for TableRow {
    fn render(self, _window: &mut Window, _cx: &mut App) -> impl IntoElement {
        let change_color = if self.data.change >= 0.0 { green() } else { red() };
        let change_sign = if self.data.change >= 0.0 { "+" } else { "" };

        div()
            .flex()
            .py(px(4.))
            .border_b_1()
            .border_color(gray())
            .child(
                div().flex_grow(1.).px(px(8.)).child(self.data.symbol.clone())
            )
            .child(
                div().flex_grow(1.).px(px(8.))
                    .child(format!("${:.2}", self.data.price))
            )
            .child(
                div().flex_grow(1.).px(px(8.))
                    .text_color(change_color)
                    .child(format!("{}{:.2}%", change_sign, self.data.change))
            )
            .child(
                div().flex_grow(1.).px(px(8.))
                    .child(format_number(self.data.volume))
            )
            .child(
                div().flex_grow(1.).px(px(8.))
                    .child(format!("${:.2}B", self.data.market_cap / 1e9))
            )
    }
}
```

`RenderOnce` vs `Render`:
- `Render` 用于 View（可变借用 `&mut self`，跨帧存在）
- `RenderOnce` 用于组件（消费 `self`，每帧重建，适合轻量级、无状态的 UI 片段）

## 步骤四：自定义滚动条

```rust
// 在 UniformList 外部渲染滚动条
fn render_scrollbar(
    &self,
    total_height: Pixels,
    viewport_height: Pixels,
    scroll_offset: Pixels,
    cx: &mut Context<Self>,
) -> impl IntoElement {
    let thumb_height = (viewport_height / total_height) * viewport_height;
    let thumb_top = (scroll_offset / total_height) * viewport_height;

    div()
        .absolute()
        .right(px(2.))
        .top(px(0.))
        .w(px(8.))
        .h(viewport_height)
        .bg(gray().fade_out(0.8))
        .rounded(px(4.))
        .child(
            div()
                .absolute()
                .top(thumb_top)
                .w(px(8.))
                .h(thumb_height)
                .bg(gray())
                .rounded(px(4.))
                .cursor(CursorStyle::PointingHand)
                .on_drag(cx.listener(|this, event: &DragMoveEvent, cx| {
                    // 拖拽滚动条 → 更新 scroll_offset
                    let ratio = event.position.y / viewport_height;
                    let new_offset = ratio * total_height;
                    this.scroll_to(new_offset, cx);
                }))
        )
}
```

## 步骤五：列排序交互

```rust
fn header_cell(
    &self,
    label: &str,
    column: SortColumn,
    cx: &mut Context<Self>,
) -> impl IntoElement {
    let is_active = self.sort_column == column;
    let arrow = if is_active {
        if self.sort_ascending { " ▲" } else { " ▼" }
    } else {
        ""
    };

    div()
        .flex_grow(1.)
        .px(px(8.))
        .py(px(6.))
        .font_weight(if is_active { FontWeight::BOLD } else { FontWeight::NORMAL })
        .cursor(CursorStyle::PointingHand)
        .child(format!("{}{}", label, arrow))
        .on_click(cx.listener(move |this, _event: &ClickEvent, cx| {
            if this.sort_column == column {
                this.sort_ascending = !this.sort_ascending;
            } else {
                this.sort_column = column;
                this.sort_ascending = true;
            }
            cx.notify();
        }))
}
```

## 步骤六：数据排序

```rust
impl DataTable {
    fn sorted_rows(&self) -> Vec<&StockRow> {
        let mut sorted: Vec<&StockRow> = self.rows.iter().collect();
        match self.sort_column {
            SortColumn::Symbol => {
                sorted.sort_by(|a, b| a.symbol.cmp(&b.symbol));
            }
            SortColumn::Price => {
                sorted.sort_by(|a, b| a.price.partial_cmp(&b.price).unwrap());
            }
            SortColumn::Change => {
                sorted.sort_by(|a, b| a.change.partial_cmp(&b.change).unwrap());
            }
            SortColumn::Volume => {
                sorted.sort_by(|a, b| a.volume.cmp(&b.volume));
            }
            SortColumn::MarketCap => {
                sorted.sort_by(|a, b| a.market_cap.partial_cmp(&b.market_cap).unwrap());
            }
        }
        if !self.sort_ascending {
            sorted.reverse();
        }
        sorted
    }
}
```

排序每次在 `render()` 中执行——对于 10,000 行来说 O(n log n) 是可接受的。如果数据量更大（100K+），你需要在数据变更时预排序（在 Entity 的 `update` 中排序），而不是在 `render()` 中。

## 性能分析

| 操作 | UniformList | 普通 List |
|------|------------|----------|
| **render() 调用次数** | ~30（可见行） | 10,000（所有行） |
| **Element 分配** | ~30 份 | 10,000 份 |
| **Taffy 布局计算** | ~30 个节点 | 10,000 个节点 |
| **内存占用** | ~30 个 Drawable | 10,000 个 Drawable |

这就是为什么对于大数据集，必须使用 `UniformList`。

## 关键收获

1. **UniformList 只渲染可见行**：通过 `item_height` 计算可见范围，大福减少 Element 数量。
2. **RenderOnce 用于轻量级组件**：`TableRow` 每帧重新创建，不需要跨帧状态。
3. **滚动条手动实现**：drag 事件更新 scroll_offset，UniformList 自动跟随。
4. **排序在 Entity 状态中管理**：`sort_column` + `sort_ascending` 保存在 `DataTable` 中，每次点击更新并 `notify()`。
