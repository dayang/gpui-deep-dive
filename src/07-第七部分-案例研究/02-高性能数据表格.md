# 第 20 章：手写一个高性能数据表格

本章基于 `examples/data_table.rs`（474 行），讲解如何构建一个支持 10,000 行数据、自定义排序、滚动条拖拽的数据表格。

> 源文件参考：`examples/data_table.rs`、`src/elements/uniform_list.rs`

## UniformList：高性能列表的秘密

普通 `List` 会为每个可见元素创建完整 Element——O(n) 渲染。`UniformList` 利用**所有行高度相同**的约束，只为可见行创建元素。

```rust
// src/elements/uniform_list.rs:22-29 — 真实 API
pub fn uniform_list<R>(
    id: impl Into<ElementId>,
    item_count: usize,
    f: impl 'static + Fn(Range<usize>, &mut Window, &mut App) -> Vec<R>,
) -> UniformList
```

三个参数：
1. **`id`**：ElementId，用于跨帧状态持久化。
2. **`item_count`**：总数据行数。
3. **`f`**：渲染闭包——接收 `(Range<usize>, &mut Window, &mut App)`，返回 `Vec<R>`（`R` 需实现 `IntoElement`）。

**不需要传入 View handle**（早期文档可能误导）。状态通过 `id` 的 element state 自动管理。

## 步骤二：UniformList 的渲染

```rust
impl Render for DataTable {
    fn render(&mut self, _window: &mut Window, cx: &mut Context<Self>) -> impl IntoElement {
        let sorted_rows: Vec<&StockRow> = self.sorted_rows();
        let count = sorted_rows.len();

        div()
            .flex_col()
            .size_full()
            .child(
                // 表头
                div().flex()
                    .child(self.header_cell("Symbol", SortColumn::Symbol, cx))
                    .child(self.header_cell("Price", SortColumn::Price, cx))
                    // ...
            )
            .child(
                uniform_list(
                    "data_rows",   // ElementId——跨帧持久化滚动位置
                    count,
                    move |range, _window, cx| {
                        // range = 可见行索引范围
                        // cx: &mut App
                        range.map(|i| {
                            TableRow::new(sorted_rows[i].clone())
                                .into_any_element()
                        }).collect()
                    },
                )
                .flex_grow(1.)
            )
    }
}
```

## 步骤三：TableRow 作为 RenderOnce 组件

```rust
#[derive(IntoElement)]
struct TableRow {
    data: StockRow,
}

impl RenderOnce for TableRow {
    fn render(self, _window: &mut Window, _cx: &mut App) -> impl IntoElement {
        let change_color = if self.data.change >= 0.0 { green() } else { red() };

        div()
            .flex()
            .py(px(4.))
            .border_b_1()
            .border_color(gray())
            .child(div().flex_grow(1.).px(px(8.)).child(self.data.symbol.clone()))
            .child(div().flex_grow(1.).px(px(8.)).child(format!("${:.2}", self.data.price)))
            .child(
                div().flex_grow(1.).px(px(8.))
                    .text_color(change_color)
                    .child(format!("{:.2}%", self.data.change))
            )
            // ...
    }
}
```

`RenderOnce` 消费 `self`，每帧重建——适合轻量级、无状态的 UI 片段。

## 步骤五：列排序交互

```rust
fn header_cell(
    &self,
    label: &str,
    column: SortColumn,
    cx: &mut Context<Self>,
) -> impl IntoElement {
    let arrow = if self.sort_column == column {
        if self.sort_ascending { " ▲" } else { " ▼" }
    } else { "" };

    div()
        .flex_grow(1.)
        .px(px(8.))
        .py(px(6.))
        .cursor(CursorStyle::PointingHand)
        .child(format!("{}{}", label, arrow))
        .on_click(cx.listener(move |this, _: &ClickEvent, _window, cx| {
            if this.sort_column == column {
                this.sort_ascending = !this.sort_ascending;
            } else {
                this.sort_column = column;
                this.sort_ascending = true;
            }
            cx.notify();
        }))
}
```

排序在 `render()` 中执行——对 10,000 行来说 O(n log n) 可接受。如果数据量更大（100K+），应在数据变更时预排序。

## 性能分析

| 操作 | UniformList | 普通 List |
|------|------------|----------|
| **render() 调用次数** | ~30（可见行） | 10,000（所有行） |
| **Element 分配** | ~30 份 | 10,000 份 |
| **Taffy 布局计算** | ~30 个节点 | 10,000 个节点 |

## 关键收获

1. **`uniform_list(id, count, fn)` 只有三个参数**——不需要传 view handle。
2. **渲染闭包签名**：`Fn(Range<usize>, &mut Window, &mut App) -> Vec<R>`。
3. **滚动位置自动持久化**——通过 `id` 的 element state 在 Frame 间迁移。
4. **`RenderOnce` 用于无状态行组件**——每帧新建，消费 `self`。
